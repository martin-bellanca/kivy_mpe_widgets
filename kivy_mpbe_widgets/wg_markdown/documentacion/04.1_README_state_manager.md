# State Manager Pattern - Implementación

## 📋 Descripción

Implementación del patrón **State Manager** para centralizar la gestión de estados de las líneas del documento markdown, eliminando la fragmentación del estado entre múltiples componentes.

## 🎯 Objetivos

- ✅ **Single Source of Truth** - Un solo lugar para el estado
- ✅ **Inmutabilidad** - Estados no se modifican, se reemplazan
- ✅ **Observabilidad** - Sistema de notificación de cambios
- ✅ **Validación** - Invariantes del estado validables
- ✅ **Trazabilidad** - Historial opcional de cambios

## 🏗️ Arquitectura

### Clases Principales

#### 1. `LineState` (Dataclass Inmutable)

Estado de una línea individual:

```python
@dataclass(frozen=True)
class LineState:
    index: int                          # Índice en la lista
    selected: bool = False              # ¿Seleccionada?
    active: bool = False                # ¿Activa? (solo una)
    editing: bool = False               # ¿En modo edición?
    hotlight: bool = False              # ¿Mouse encima?
    visible: bool = True                # ¿Visible? (filtros)
    cursor_pos: Tuple[int, int] = (0,0) # Posición cursor
```

**Características:**
- **Inmutable** (`frozen=True`) - No se puede modificar
- Método `with_changes(**kwargs)` para crear nuevos estados
- Representación legible con `__repr__`

#### 2. `StateChangeEvent`

Evento de notificación de cambio:

```python
class StateChangeEvent:
    index: int              # Línea que cambió
    old_state: LineState    # Estado anterior
    new_state: LineState    # Estado nuevo

    @property
    def changed_attributes(self) -> Set[str]:
        # Retorna atributos que cambiaron
```

#### 3. `DocumentStateManager`

Gestor centralizado de estados:

```python
class DocumentStateManager:
    # Almacenamiento
    _states: Dict[int, LineState]
    _active_index: Optional[int]
    _selected_indices: Set[int]

    # Observers
    _observers: List[Callable]

    # Métodos principales
    def get_state(index) -> LineState
    def update_state(index, **changes)
    def activate_line(index, enter_edit_mode, cursor_pos)
    def select_line(index, multi)
    def subscribe(observer)
```

### Sistema de Observadores

```python
# Definir observador
def on_state_changed(event: StateChangeEvent):
    print(f"Línea {event.index} cambió:")
    print(f"  Atributos: {event.changed_attributes}")
    print(f"  Nuevo estado: {event.new_state}")

# Suscribirse
manager.subscribe(on_state_changed)

# Ahora cada cambio de estado notificará
manager.activate_line(5)
# Output:
# Línea 5 cambió:
#   Atributos: {'active', 'selected'}
#   Nuevo estado: LineState[5](SEL|ACT)
```

### Paso 2: Inicializar estados al cargar documento

```python
def populate_from_md_lines(self, md_lines):
    """Carga documento."""
    self._md_lines = md_lines

    # ✅ Inicializar estados
    self.state_manager.initialize_states(len(md_lines))

    self._refresh_recycleview_data()
```

### Paso 3: Incluir estado en RecycleView.data

```python
def _refresh_recycleview_data(self):
    """Actualiza RecycleView.data."""
    self.data = []

    for index, md_line in enumerate(self._md_lines):
        # ✅ Obtener estado del manager
        state = self.state_manager.get_state(index)

        self.data.append({
            'md_line': md_line,
            'index': index,
            'state': state,  # ✅ Estado completo
            'show': {...},
            'themed': {...}
        })
```

### Paso 4: Manejar cambios de estado

```python
def _on_line_state_changed(self, event: StateChangeEvent):
    """Callback de cambios de estado."""
    # Obtener widget si es visible
    widget = self._get_widget_at_index(event.index)

    if widget:
        # ✅ Aplicar nuevo estado
        widget.apply_state(event.new_state)
```

### Paso 5: Usar StateManager en eventos

```python
def handle_touch_left_up_event(self, index, view, touch):
```

## 🔧 Integración con MDDocumentLineEditor

### Método `apply_state()`

Crear método para aplicar estado completo:

```python
class MDDocumentLineEditor(RecycleDataViewBehavior, ThemeWidget):

    def apply_state(self, state: LineState):
        """Aplica estado completo al widget."""

        # Actualizar selección
        if state.selected != self.graphic_select.is_visible():
            self.graphic_select.show(state.selected)

        # Actualizar activación
        if state.active != self.graphic_active.is_visible():
            self.graphic_active.show(state.active)

        # Actualizar hotlight
        if state.hotlight != self.graphic_hotlight.is_visible():
            self.graphic_hotlight.show(state.hotlight)

        # Actualizar modo edición
        if state.editing != self.wg_line_editor.mode_editor:
            self.wg_line_editor.show_editor(
                state.editing,
                cursor=state.cursor_pos if state.editing else None
            )
```

### Actualizar `refresh_view_attrs()`

```python
def refresh_view_attrs(self, rv, index, data):
    """Actualiza widget al reciclar."""
    self.index = index

    # ✅ Obtener estado del data
    state = data.get('state')

    if state:
        # Aplicar estado completo
```

## 📊 Beneficios de la Implementación

### 1. Centralización

**Antes:**
```python
# Estado disperso en 3+ lugares
class MDDocumentEditor:
    _active_index = -1
    _selected_indexs = []
    _mode_editor = False

class MDDocumentLineEditor:
    hotlight = BooleanProperty()
    di_state = DataState()

class DataState:
    selected = bool
    active = bool
    # ...
```

**Después:**
```python
# Un solo lugar
class MDDocumentEditor:
    state_manager = DocumentStateManager()

# Acceso unificado
state = editor.state_manager.get_state(index)
```

### 2. Consistencia

**Antes:**
```python
# Sincronización manual propensa a errores
self._active_index = 5
self.data[5]['state'].active = True
self.data[5]['selected'] = True
# ¿Actualizamos _selected_indexs?
# ¿Desactivamos línea anterior?
```

**Después:**
```python
# Automático y consistente
manager.activate_line(5)
# ✅ Desactiva anterior
# ✅ Actualiza selección
# ✅ Notifica cambios
```

### 3. Trazabilidad

**Antes:**
```python
# ¿Cuándo cambió el estado?
# ¿Quién lo cambió?
# Difícil de rastrear
```

**Después:**
```python
# Historial completo (opcional)
manager = DocumentStateManager(enable_history=True)

# Cada cambio registrado
history = manager.get_history()
for event in history:
    print(event)  # índice, qué cambió, cuándo
```

### 4. Validación

**Antes:**
```python
# Estado puede ser inconsistente
# Sin forma de detectar
```

**Después:**
```python
# Validación de invariantes
if manager.validate_invariants():
    print("Estado válido ✅")
else:
    print("Estado inválido ❌")

# Detecta:
# - Múltiples líneas activas
# - Línea activa no seleccionada
# - Índices desincronizados
```

### 5. Testing

**Antes:**
```python
# Difícil testear sin UI
# Estado acoplado a widgets
```

**Después:**
```python
# Testeo independiente de UI
manager = DocumentStateManager()
manager.activate_line(5)
assert manager.get_active_index() == 5
```

## 📝 Notas de Implementación

### Índices y Operaciones de Documento

Cuando se inserta/elimina una línea, los índices deben ajustarse:

```python
# Insertar línea en posición 5
# Todos los índices >= 5 se incrementan
manager.shift_indices(start_index=5, delta=1)

# Eliminar línea en posición 8
# Todos los índices >= 9 se decrementan
manager.shift_indices(start_index=9, delta=-1)
```

### Performance

- **Complejidad de operaciones:**
  - `get_state()`: O(1)
  - `update_state()`: O(1) + O(n_observers)
  - `select_range()`: O(end - start)
  - `clear_selection()`: O(n_selected)

- **Memoria:**
  - Por línea: ~100 bytes (LineState)
  - Total para 10,000 líneas: ~1 MB

### Thread Safety

**⚠️ IMPORTANTE:** El StateManager NO es thread-safe.

Si se usa en contexto multi-thread, agregar locks:

```python
import threading

class ThreadSafeStateManager(DocumentStateManager):
    def __init__(self, **kwargs):
        super().__init__(**kwargs)
        self._lock = threading.Lock()

    def update_state(self, index, **changes):
        with self._lock:
            super().update_state(index, **changes)
```
