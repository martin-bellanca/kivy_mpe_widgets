# Service Layer Pattern - Implementación Fase 2

## 📋 Descripción

Implementación de la **Capa de Servicios (Service Layer)** que encapsula la lógica de negocio del editor markdown, separándola de la capa de presentación (UI widgets).

Este patrón complementa el **State Manager** (Fase 1) proporcionando servicios especializados para operaciones específicas del dominio.

## 🎯 Objetivos Cumplidos

- ✅ **Separación de Responsabilidades** - Lógica de negocio separada de la UI
- ✅ **Servicios Especializados** - Cada servicio tiene una responsabilidad única
- ✅ **Testabilidad** - Servicios independientes fáciles de testear
- ✅ **Reusabilidad** - Servicios utilizables desde diferentes partes de la aplicación
- ✅ **Mantenibilidad** - Cambios en lógica de negocio aislados en servicios

## 🏗️ Arquitectura de Servicios

### Diagrama de Capas

```
┌─────────────────────────────────────────────────────┐
│           PRESENTATION LAYER (UI)                    │
│  MDDocumentEditor, MDDocumentLineEditor, Widgets    │
└──────────────────┬──────────────────────────────────┘
                   │ usa
                   ▼
┌─────────────────────────────────────────────────────┐
│           SERVICE LAYER                              │
│  LineService │ SelectionService │ NavigationService │
└──────────────────┬──────────────────────────────────┘
                   │ usa
                   ▼
┌─────────────────────────────────────────────────────┐
│           STATE MANAGEMENT                           │
│          DocumentStateManager                        │
└──────────────────┬──────────────────────────────────┘
                   │ gestiona
                   ▼
┌─────────────────────────────────────────────────────┐
│           DOMAIN MODEL                               │
│         MDDocument, MDLine                           │
└─────────────────────────────────────────────────────┘
```

### Principios de Diseño

1. **Single Responsibility**: Cada servicio tiene una responsabilidad única
2. **Dependency Injection**: Servicios reciben dependencias en constructor
3. **Interface Segregation**: Métodos públicos bien definidos
4. **Low Coupling**: Servicios comunicados a través de StateManager
5. **High Cohesion**: Operaciones relacionadas agrupadas en mismo servicio

## 🔧 Servicios Implementados

### 1. LineService

**Responsabilidad**: Operaciones sobre líneas individuales del documento.

**Dependencias**:
- `DocumentStateManager`: Para gestión de estado
- `md_lines: List[MDLine]`: Lista de líneas del documento

**Operaciones Principales**:

```python
class LineService:
    # Activación y Edición
    def activate_line(index, enter_edit, cursor_pos) -> bool
    def deactivate_current_line(save_changes) -> bool
    def toggle_edit_mode() -> bool

    # Movimiento
    def move_line_up(index) -> Optional[int]
    def move_line_down(index) -> Optional[int]

    # Inserción
    def insert_line_below(index, text, line_type) -> int
    def insert_line_above(index, text, line_type) -> int

    # Eliminación
    def delete_line(index) -> bool

    # Consultas
    def is_line_editable(index) -> bool
    def get_line_type(index) -> MD_LINE_TYPE
    def get_line_text(index) -> str
    def update_line_text(index, new_text) -> bool
```

**Características Especiales**:
- Valida tipos de línea editables vs no editables
- Actualiza automáticamente referencias `prev_line`/`next_line`
- Sincroniza con StateManager al mover/insertar/eliminar líneas
- Actualiza `num_line` automáticamente

**Tipos Editables**:
- TEXT, TITLE, HEAD_TITLE, LIST, ORDER_LIST, TASK, TODO, BLOCKQUOTE

**Tipos NO Editables**:
- SEPARATOR, UNDERLINE_TITLE, TABLE, IMAGEN, CODE, START_CODE, END_CODE

### 2. SelectionService

**Responsabilidad**: Gestión de selección de líneas (simple y múltiple).

**Dependencias**:
- `DocumentStateManager`: Para gestión de estado de selección
- `md_lines: List[MDLine]`: Lista de líneas del documento

**Operaciones Principales**:

```python
class SelectionService:
    # Selección Básica
    def select_single(index) -> bool           # Selección simple (limpia anteriores)
    def select_multi(index) -> bool            # Selección múltiple (agrega)
    def select_range(start, end) -> int        # Selección de rango
    def toggle_selection(index) -> bool        # Alternar selección

    # Deselección
    def unselect(index) -> bool                # Deseleccionar línea específica
    def clear_selection() -> int               # Limpiar toda la selección

    # Selección Inteligente
    def select_all() -> int                    # Seleccionar todas las líneas
    def select_visible() -> int                # Seleccionar solo visibles
    def invert_selection() -> int              # Invertir selección actual

    # Extensión
    def extend_selection_to(target_index) -> int  # Extender desde activa

    # Consultas
    def is_selected(index) -> bool
    def get_selected_indices() -> Set[int]
    def get_selected_count() -> int
    def has_selection() -> bool
    def get_selected_lines() -> List[MDLine]
    def get_selection_range() -> tuple[int, int]
```

**Características Especiales**:
- Soporta selección tipo "Shift+Click" con `extend_selection_to()`
- Maneja selección tipo "Ctrl+A" con `select_all()`
- Respeta visibilidad de líneas en `select_visible()`
- Métodos retornan contadores para feedback al usuario

### 3. NavigationService

**Responsabilidad**: Navegación estructurada por el documento markdown.

**Dependencias**:
- `DocumentStateManager`: Para obtener línea activa
- `md_lines: List[MDLine]`: Lista de líneas del documento

**Operaciones Principales**:

```python
class NavigationService:
    # Navegación Secuencial
    def navigate_to_next_line() -> Optional[int]
    def navigate_to_previous_line() -> Optional[int]
    def navigate_to_first_line() -> Optional[int]
    def navigate_to_last_line() -> Optional[int]

    # Navegación por Títulos
    def navigate_to_next_title(from_index) -> Optional[int]
    def navigate_to_previous_title(from_index) -> Optional[int]
    def navigate_to_parent_title(from_index) -> Optional[int]
    def navigate_to_next_sibling_title(from_index) -> Optional[int]

    # Navegación por Tipo
    def navigate_to_next_of_type(line_type, from_index) -> Optional[int]
    def navigate_to_previous_of_type(line_type, from_index) -> Optional[int]

    # Búsqueda
    def find_line_by_text(search_text, case_sensitive, from_index) -> Optional[int]
    def find_line_by_predicate(predicate, from_index) -> Optional[int]
    def find_all_of_type(line_type) -> List[int]

    # Navegación por Visibilidad
    def navigate_to_next_visible(from_index) -> Optional[int]
    def navigate_to_previous_visible(from_index) -> Optional[int]

    # Utilidades
    def get_line_at_index(index) -> Optional[MDLine]
    def get_visible_count() -> int
```

**Características Especiales**:
- Aprovecha estructura jerárquica de títulos con `get_title_parent()`
- Búsqueda case-sensitive/insensitive
- Búsqueda por predicado (lambda personalizada)
- Navegación respeta filtros de visibilidad
- Métodos retornan índices, no cambian estado (idempotentes)

## 🚀 Uso de los Servicios

### Integración en MDDocumentEditor

```python
class MDDocumentEditor(RecycleView):

    def __init__(self, **kwargs):
        super().__init__(**kwargs)

        # ✅ Crear StateManager
        self.state_manager = DocumentStateManager()

        # ✅ Crear Servicios (después de tener md_lines)
        self.line_service = None
        self.selection_service = None
        self.navigation_service = None

        # ✅ Suscribirse a cambios de estado
        self.state_manager.subscribe(self._on_line_state_changed)

    def populate_from_md_lines(self, md_lines):
        """Carga documento."""
        self._md_lines = md_lines

        # ✅ Inicializar estados
        self.state_manager.initialize_states(len(md_lines))

        # ✅ Crear servicios con dependencias
        self.line_service = LineService(
            self.state_manager,
            self._md_lines
        )

        self.selection_service = SelectionService(
            self.state_manager,
            self._md_lines
        )

        self.navigation_service = NavigationService(
            self.state_manager,
            self._md_lines
        )

        self._refresh_recycleview_data()
```

```python
def handle_touch_left_up_event(self, index, view, touch):
    # ✅ Lógica delegada a servicio
    cursor = view.wg_line_editor.md_editor.get_cursor_from_xy(
        *view.to_local(*touch.pos)
    )

    success = self.line_service.activate_line(
        index,
        enter_edit=True,
        cursor_pos=cursor
    )

    if not success:
        self.show_message("Esta línea no se puede editar")
```

### Ejemplo 3: Navegación por Teclado

```python
def on_keyboard_arrow_down(self):
    """Flecha abajo."""
    next_index = self.navigation_service.navigate_to_next_line()

    if next_index is not None:
        self.line_service.activate_line(next_index)
        self.ensure_visible(next_index)

def on_keyboard_ctrl_down(self):
    """Ctrl+Flecha abajo - Ir a siguiente título."""
    next_title = self.navigation_service.navigate_to_next_title()

    if next_title is not None:
        self.line_service.activate_line(next_title)
        self.scroll_to_line(next_title)

def on_keyboard_alt_up(self):
    """Alt+Flecha arriba - Mover línea arriba."""
    new_index = self.line_service.move_line_up()

    if new_index is not None:
        self._refresh_recycleview_data()
        self.ensure_visible(new_index)
```

### Ejemplo 4: Operaciones de Edición

```python
def on_keyboard_enter(self):
    """Enter - Insertar línea abajo."""
    active_index = self.state_manager.get_active_index()

    if active_index is not None:
        new_index = self.line_service.insert_line_below(
            index=active_index,
            text="",
            line_type=MD_LINE_TYPE.TEXT
        )

        # Refresh UI
        self._refresh_recycleview_data()

        # Activar nueva línea
        self.line_service.activate_line(
            new_index,
            enter_edit=True
        )

def on_keyboard_delete(self):
    """Delete - Eliminar líneas seleccionadas."""
    selected = self.selection_service.get_selected_indices()

    # Eliminar de mayor a menor para no alterar índices
    for index in sorted(selected, reverse=True):
        self.line_service.delete_line(index)

    # Refresh
    self._refresh_recycleview_data()
    self.show_status(f"{len(selected)} líneas eliminadas")
```

## 📊 Beneficios de la Implementación

### 1. Separación de Responsabilidades

**Antes**:
```python
class MDDocumentEditor:
    def handle_click(self, index):
        # 100+ líneas de código mezclando:
        # - Lógica de negocio
        # - Manipulación de UI
        # - Validaciones
        # - Actualizaciones de estado
        # Difícil de mantener y testear
```

**Después**:
```python
class MDDocumentEditor:
    def handle_click(self, index):
        # 5 líneas claras
        cursor = self.get_cursor_from_touch(touch)
        success = self.line_service.activate_line(index, enter_edit=True, cursor_pos=cursor)
        if not success:
            self.show_error_message("Línea no editable")
        # Fácil de leer, mantener y testear
```

### 2. Testabilidad

**Antes**:
- Necesitabas instanciar toda la UI de Kivy para testear lógica
- Tests lentos (requieren framework gráfico)
- Difícil aislar componentes

**Después**:
- Tests unitarios sin dependencias de UI
- Tests rápidos (0.06 segundos para 89 tests)
- Fácil aislar y mockear dependencias

```python
# Test simple y rápido
def test_activate_line():
    state_manager = DocumentStateManager()
    md_lines = [FakeMDLine(f"Line {i}", i) for i in range(10)]
    state_manager.initialize_states(10)

    service = LineService(state_manager, md_lines)
    success = service.activate_line(3)

    assert success
    assert state_manager.get_active_index() == 3
```

## 🔍 Patrones de Diseño Aplicados

### 1. Service Layer Pattern
Encapsular lógica de negocio en servicios reutilizables.

### 2. Dependency Injection
Servicios reciben dependencias en constructor.

```python
service = LineService(
    state_manager=self.state_manager,  # inyectado
    md_lines=self._md_lines            # inyectado
)
```

### 3. Single Responsibility Principle
Cada servicio tiene una responsabilidad única:
- LineService: Operaciones de líneas
- SelectionService: Selección
- NavigationService: Navegación

### 4. Interface Segregation Principle
Métodos públicos bien definidos, implementación privada oculta.

```python
# Público
def activate_line(self, index):
    ...

# Privado
def _is_valid_index(self, index):
    ...
```

### 5. Command Pattern (implícito)
Cada método del servicio es como un comando ejecutable.

```python
# Comandos
line_service.move_line_up()
line_service.delete_line()
selection_service.select_all()
# Fácil de deshacer/rehacer en Fase 4
```

## 🎓 Lecciones Aprendidas

1. **Separar lógica de negocio de UI es crucial**
   - Facilita testing
   - Mejora mantenibilidad
   - Reduce acoplamiento

2. **Servicios pequeños y focalizados son mejores**
   - Más fáciles de entender
   - Más fáciles de testear
   - Más fáciles de reutilizar

3. **Dependency Injection hace código testeable**
   - Permite mockear dependencias
   - Facilita tests unitarios
   - Mejora flexibilidad

4. **Tests son inversión, no costo**
   - Detectan bugs temprano
   - Documentan comportamiento esperado
   - Dan confianza para refactorizar

5. **StateManager + Services = Arquitectura sólida**
   - StateManager para el "qué" (estado)
   - Services para el "cómo" (operaciones)
   - UI solo para el "dónde" (presentación)
